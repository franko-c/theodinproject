HTML Boilerplate

An HTML boilerplate, or just a boilerplate, is a block of code, a bit like a template, that is designed to get you up and running with your website, in as little time as possible.

A boilerplate allows a web developer, no matter their level of experience, to jump right into creating a website, without the need to worry about the initial setting up of the HTML elements required for a webpage to work, the following elements are the most often included in a boilerplate:

  • !DOCTYPE
  • html
  • head
  • body

Generally, in the context of HTML, a boilerplate will, at the very least, include all the necessary tags required for your webpage to pass the W3C validation checks, assuming the additional changes you make are also in line with their standards.

Ramblings on Boilerplates

HTML has been around long enough that it Just Makes Sense to have a quick, easy, reusable and indeed, customisable way to get a website - as barebones as it may be - up and running.

It may come as no surprise then, that the concept of a boilerplate exists for CSS files, too.

The most popular CSS boilerplates come in two flavours, and they are quite different in their approach at providing you with a starting point.

CSS Reset

The original boilerplate, commonly known as a CSS Reset, aims to remove as much of the default styles applied by the browser as possible. In practice, this allows, and even demands, that the developer be as explicit as possible in their approach when styling.However, it also means that any default styles that you might have found beneficial, are now gone.

Normalize.css

Normalize.css is a CSS library that aims to make the default styling of HTML elements consistent across different web browsers. It does this by: - Correcting browser inconsistencies in how elements are rendered - Normalizing base styles like font, line-height, and margins/padding - Improving accessibility of elements like links, SVGs, and form controls - Modernizing defaults for newer HTML5 elements
